Purpose

POI-5026 is the authoritative register of all Point of Interest (POI) categories for trackaroo® Phase 1. It defines the complete Phase 1 POI set: categories included, display rules, archetype relevance, group-level toggle model, search capability, data source model, offline bundle obligations, PCR framework intersection, and phase boundary classification.

This document was drafted under the working reference PCR-5026. Before issue, the Project Director directed that it be renamed POI-5026 to eliminate potential confusion with the Point Condition Report (PCR) framework governed by OSM-5026. The working reference PCR-5026 is retired and must not appear in any document, system, code comment, or communication.

This document also corrects a framing error present in earlier drafts and in CRG-5026 Section 4.3, which caused camping, caravanning, and shelter location data to be presented as though it occupied the same exclusion space as lifestyle discovery features. Camping, caravanning, and shelter location data -- including user reporting of such locations via the PCR framework -- is core to trackaroo® purpose and to facilitating safe travel. This position is stated as a governing principle in the Content section and as a mandatory delivery requirement in Constraints.

CONSTRAINT

POI-5026 does not introduce new Phase 1 scope. Every POI category listed as Phase 1 Active in this document is already implied, referenced, or required by existing Phase 1 authority documents. This register clarifies, consolidates, and formally governs what was already within scope. It does not constitute a variation, scope extension, or basis for any commercial claim by the delivery partner. Any POI category not listed as Phase 1 Active is explicitly out of Phase 1 scope. Any future Phase 1 addition requires a formal Project Director-approved scope variation recorded in CAR-5026.

Constraints

The following are absolute restrictions that apply unconditionally to all Phase 1 POI build and design decisions. No departure from these constraints is permitted without written Project Director approval.

CONSTRAINT

CAM-01 through CAM-05 (camping, caravanning, and shelter POIs) are mandatory Phase 1 Active deliverables. They must not be reduced, omitted, deprioritised, or treated as optional in any build or design decision. The Camp activity context is Phase 1 Active (TAA-5126). Authority: CRG-5026-A1; TAA-5126; UXS-5726.

CONSTRAINT

All Phase 1 POI data must be available in a fully offline state. If the device is in airplane mode, all POI display, search, group toggle, and navigation destination functions must remain fully operational. POI categories requiring live connectivity for any core function are not eligible for Phase 1 Active status. Authority: UXS-5726; FSD-5126.

CONSTRAINT

POI display, search results, and group toggle defaults must not use AI inference, adaptive logic, personalisation scoring, telemetry-derived weighting, or ML-based content selection. All POI behaviours are deterministic. Violation constitutes a Survival Core governance breach. Authority: UXS-5726.

CONSTRAINT

Photo-enabled POIs are deferred to Phase 2. No POI category in Phase 1 may accept, display, or store user-submitted photographs. Phase 2 scaffolding for photo capability must be inert and non- executable, confirmed by QA. Authority: PSB-5026.

Content

6.1 POI System Architecture

6.1.1 Layer Independence POI markers are a distinct map layer, independent of the overlay systems governed by OSM-5026. PCR markers (track-point and POI-location condition reports) and POI markers (static infrastructure records) must never be merged, share icon systems, or modify each other's display state. OSM-5026 LIR-04 and LIR-05 govern layer separation. A HazTrack™ event does not alter POI display state. A PCR submission does not alter POI display state.

CONSTRAINT

PCR submissions must not modify POI display state, icon, label, or prominence. POI data is static bundle data. PCR data is user-submitted condition data. Merging these systems constitutes a governance breach against OSM-5026 LIR-04 and LIR-05. This constraint applies to all POI types including campsite, shelter, and caravan park POIs.

6.1.2 Navigation Destination Integration A POI must be selectable as a navigation destination from the map view or the POI detail panel. A 'Navigate here' action sets the POI coordinates as the active navigation destination without requiring connectivity and without overwriting the active BackTrack™ breadcrumb trail. Minimum touch target: 44x44pt. Recommended for Survival Core controls: 60x60pt. Authority: CQR-5026-CR04 Q1; FSD-5126; UXS-5726.

6.1.3 Data Classification Phase 1 POI data is read-only reference data sourced from the downloaded Mapbox SDK + OSM vector tile bundle. It is not user-generated, user-modified, or cloud-synced in Phase 1. User-saved POI favourites, if implemented in Phase 1, are Firebase-permitted but must not affect Survival Core behaviour. Photo-enabled POI data is not permitted in Phase 1. Authority: CDG-5126; PSB-5026.

6.1.4 PCR Availability at POI Locations PCR submission is available at all POI types in Phase 1 -- not limited to named track segment points. Campsite POIs (CAM-01 through CAM-05), shelter POIs, caravan park POIs, and facility POIs (FAC-01 through FAC-06) are all eligible PCR submission locations. Users can submit PCR-CLO, PCR-INF, PCR- WTR, PCR-OBS, and PCR-HAZ reports at any POI location. No separate campsite condition reporting mechanism is required or permitted. The PCR framework governed by OSM-5026 is the sole user-submitted condition reporting mechanism across all map point types.

6.1.5 WFD-5126 Wireframe Gate

RELEASE GATE

The following wireframe states are required in WFD-5126 before any POI system development may commence: (1) POI marker on map -- default and tapped state; (2) POI detail panel -- all Phase 1 Active category types; (3) POI search entry point and results list; (4) POI group filter / toggle panel - - all ten display groups with toggle controls and archetype defaults indicated; (5) 'Navigate here' flow from POI detail panel; (6) POI marker -- glove-compatible touch target validation state (minimum 44x44pt); (7) Empty state -- no POIs in downloaded region for a given group. No development may commence without WFD-5126 approval of all seven states.

6.2 POI Search Capability

POI search is a Phase 1 Active capability. It supports the Five-Question Hierarchy question 'Where am I going?' by enabling users to locate a specific POI by name or type without browsing the map visually. All search functions must operate fully offline against the downloaded regional bundle.

Search Capability Specification

Search entry point Available from the main map view via a persistent search control. Must not be

buried more than one tap from the map. Also available from the POI detail flow.

Search scope Searches the complete Phase 1 Active POI dataset within the downloaded

regional bundle. No network call required or permitted.

Search fields Matches against: POI display name; POI category display name; POI group

name. Partial string matching required. Example: 'camp' returns results across CAM-01 to CAM-05 and any POI with 'camp' in its name.

Search results Returned as a list sorted by distance from current GPS position (nearest first). Each result shows: POI name, category icon, group label, distance. Tapping a result opens the POI detail panel and centres the map on the POI.

Deterministic ranking Results must be ranked by distance only. No AI ranking, relevance scoring,

popularity weighting, or adaptive ordering. Authority: UXS-5726; Constraints Section 5.

Offline operation Search index is built from the downloaded bundle at download time or first

launch. No connectivity required at search time. Search index must survive app restart.

Search and group toggle interaction

trackaroo® | Trackaroo Systems Pty Ltd | CONFIDENTIAL Page

Search returns results across all POI groups regardless of current group toggle state. A result from a toggled-off group must still appear in search and must be navigable. Toggling off a group does not remove it from search results.

No AI or predictive input Search must not offer predictive text, AI-suggested queries, or adaptive search

history weighting. Standard autocomplete against the downloaded POI name index is permitted.

NOTE

POI search scope is limited to the downloaded regional bundle. POIs outside the downloaded region are not searchable or displayable. The vendor must implement an empty state that informs the user of this limitation and provides a clear action path to expand their regional map download. The empty state copy direction must be submitted for Project Director review as part of the WFD- 5126 wireframe approval process. Authority: WFD-5126; UXS-5726 (non-alarming language model).

6.3 POI Display Groups & Toggle Model

6.3.1 Group-Level Toggle Model POI categories are organised into ten display groups. Users toggle entire groups on or off from a dedicated filter panel. This is the primary POI visibility control. Individual category-level control is not exposed in Phase 1 -- group-level toggle is the only user-facing control.

Group-level toggle was chosen over the alternatives on the following basis:

Model Description Assessment

All on/off (binary) Single toggle for all POIs as one

group. Used by Hema 4WD Explorer (4x4 Explorer app).

trackaroo® | Trackaroo Systems Pty Ltd | CONFIDENTIAL Page

Too blunt for 55 Phase 1 Active categories across ten display groups. At high map density, all-POI display is cognitively overwhelming. Hema's implementation with 500,000 POIs is unusable at this granularity.

Group-level toggle (recommended)

User toggles functional groups (Water, Fuel, Camping etc.) on or off. Archetype preset sets group defaults.

Appropriate cognitive granularity. Matches how users think about their needs (I want water and camping visible; I don't need specialist ride POIs on this trip). Smarter than Hema; simpler than GAIA GPS.

Individual category toggle

User toggles each of 44 category codes independently.

Too granular for field use under stress. Appropriate for a desktop trip-planning tool but not for a mobile safety-oriented app. May be considered as a Phase 2 power-user settings feature.

NOTE

GAIA GPS (primary global competitor for the Bushwalker and backcountry archetype) implements individual overlay toggling through a multi-step Adventure Mode and layers menu. Its 2026 update has drawn user criticism for requiring too many taps to switch between layers in the field. trackaroo®'s group-level toggle with archetype-preset defaults delivers a simpler, faster field experience without requiring users to assemble their POI view from scratch. This is a deliberate competitive differentiator. Authority: CRG-5026.

6.3.3 Toggle Behaviour Rules

Rule Specification

Archetype preset defaults

Archetype preset sets group default states at first launch or archetype change. Default states are as declared in Section 6.3.2. Default states must not be modified by telemetry, route history, or adaptive logic.

User override The user may toggle any group on or off at any time from the filter panel. This is

always a manual, user-initiated action.

Preference persistence

User group toggle preferences persist across sessions without requiring connectivity. Preferences survive app restart.

Search override Search returns results across all groups regardless of toggle state (see Section 6.2).

Emergency group lock

Group 4 (Emergency & Safety Services) is default ON for all archetypes and must not be removable from the filter panel. The user may toggle it off manually if they choose, but the default must always be ON. No archetype preset may default this group to OFF.

Camping group protection

Group 3 (Camping, Caravanning & Shelter) is default ON for all archetypes. No archetype preset may default this group to OFF. Authority: Constraints Section 5.

No auto-suggestion The system must not prompt the user to enable a group based on location, route

context, or inferred intent.

No adaptive defaults Group toggle defaults must not change based on any telemetry or usage data. They

are fixed by archetype preset at the values declared in Section 6.3.2.

6.5 PCR Framework Intersection with POI Categories

PCR markers are visually and architecturally independent from POI markers. The following table documents the primary intersections between PCR categories and POI category codes. PCR submission is available at all POI locations (see Section 6.1.4).

PCR Category

trackaroo® | Trackaroo Systems Pty Ltd | CONFIDENTIAL Page

Relevant POI Codes

Interaction Primary Use Cases

PCR-OBS (Obstruction)

TRK-01, TRK-05, CAM-01 to CAM-04, FHT-01

PCR-OBS does not alter POI display state. Both are independently visible.

Fallen tree blocking campsite access; debris on a track junction; obstruction preventing boat ramp access.

PCR-CLO (Closure)

CAM-01 to CAM-05, ACC-01 to ACC-04

PCR-CLO may indicate a POI destination is inaccessible. The POI remains displayed. PCR provides supplementary condition data.

Campground closed by land manager; gate locked; seasonal closure; damaged hut condemned and closed.

PCR-INF (Infrastructure Damage)

CAM-05, FAC-01 to FAC-06, TRK-03, TRK-04

Infrastructure damage surfaced via PCR-INF. POI icon unchanged.

Hut or shelter roof damaged; toilet facility out of service; causeway damaged.

PCR-SRF (Surface Condition)

TRK-05, WAT-03, TRK-03, CAM-01 to CAM-04

Track surface or campsite access surface condition. POI display unaffected.

Boggy campsite access track; flooded ford; washed-out entry road.

PCR-WTR (Water Crossing)

WAT-03, TRK-03, TRK-04, FHT-01, FHT-04, CAM-01 to CAM-04

Highest-relevance intersection. Provides current crossing condition context for crossings near campsite, shelter, or water access point.

Flooded creek crossing on approach to a free camp; causeway underwater; boat ramp submerged or inaccessible due to flood.

PCR-HAZ (Localised Hazard)

EMG-06, CAM-05, ACC-03, CAM-01 to CAM-04

Hazard at or near a campsite, shelter, or access point. POI display unaffected.

Fire danger near a campsite; dangerous wildlife; unstable terrain near a hut.
