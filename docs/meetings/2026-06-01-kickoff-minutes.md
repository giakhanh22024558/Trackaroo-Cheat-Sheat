The constraints Slitigenz identified are **correct and confirmed**. PD added the following:

| # | Added constraint | Detail |
|---|---|---|
| 1 | **Brand Guidelines (issued)** | Logo usage · colour palette · typography · brand identity rules — apply to the website in full |
| 2 | **UIX-5026 §4 — Visual Register & Aesthetic Direction** | Website must reflect the same **premium outdoor, rugged-precision aesthetic register** as the mobile app. The website is a direct extension of the product experience. Must NOT feel like a generic SaaS landing page or consumer lifestyle site. |
| 3 | **UIX-5026 §5 — Quality Reference Applications** | UIX-5026 names reference apps defining the **quality floor** for visual direction. The **Design Direction Statement (Component B of the Design Intent Submission) must address** how the website design draws from + relates to those references. Website expected to meet the same quality register. |

### 7.2 Design flexibility — accepted (within §7.1 constraints)

Flexibility in **page layout · navigation structure · visual hierarchy · typography · section ordering** is **accepted**, within the constraints above.
