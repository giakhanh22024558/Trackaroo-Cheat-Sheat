# Design Decisions — Open Choices for Client Confirmation

**Purpose:** Track design decisions where the spec is silent or offers multiple paths, and we picked one to keep the proposal moving. Every row here is **negotiable** — bring to client for confirmation before vendor lock-in.

**How to use:**
- Each row = one decision point
- **Chosen** column = what's currently in the diagram / spec docs
- **Alternatives** column = other options the client might prefer
- **Trigger to revisit** = the moment this decision needs to be reconfirmed (e.g., before Discovery Gate, after vendor selection, etc.)

**Status legend:**
- 🟢 **Locked-in** — client confirmed, no change expected
- 🟡 **Provisional** — we picked one, need client confirmation
- 🔴 **Blocker** — needs decision before next milestone
