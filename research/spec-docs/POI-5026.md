## §6 Content

### §6.1 POI System Architecture

#### §6.1.1 Layer Independence
POI markers = distinct map layer, independent of OSM-5026 overlay systems. **PCR markers + POI markers must NEVER merge, share icon systems, or modify each other's display state.** OSM-5026 LIR-04 + LIR-05 govern layer separation. A HazTrack™ event does NOT alter POI display. A PCR submission does NOT alter POI display.

> **CONSTRAINT:** PCR submissions must not modify POI display state, icon, label, or prominence. POI data = static bundle data. PCR data = user-submitted condition data. Merging = governance breach against OSM-5026 LIR-04 + LIR-05. Applies to all POI types including campsite, shelter, caravan park POIs.

#### §6.1.2 Navigation Destination Integration
POI must be selectable as nav destination from map view or POI detail panel. *'Navigate here'* sets POI coords as active nav destination without connectivity + without overwriting active BackTrack™ breadcrumb. Min touch target: 44×44pt. Recommended Survival Core: 60×60pt. Authority: CQR-5026-CR04 Q1; FSD-5126; UXS-5726.

#### §6.1.3 Data Classification
Phase 1 POI data = **read-only reference data** sourced from downloaded Mapbox SDK + OSM vector tile bundle. NOT user-generated, NOT user-modified, NOT cloud-synced in Phase 1. User-saved POI favourites (if implemented) = Firebase-permitted but must not affect Survival Core. Photo-enabled POI data = not permitted Phase 1. Authority: CDG-5126; PSB-5026.

#### §6.1.4 PCR Availability at POI Locations
**PCR submission available at ALL POI types Phase 1** — not limited to named track segment points. Campsite POIs (CAM-01..05), shelter POIs, caravan park POIs, facility POIs (FAC-01..06) all eligible PCR submission locations. Users can submit PCR-CLO, PCR-INF, PCR-WTR, PCR-OBS, PCR-HAZ at any POI location. **No separate campsite condition reporting mechanism required or permitted.** PCR framework governed by OSM-5026 = sole user-submitted condition reporting mechanism across all map point types.

#### §6.1.5 WFD-5126 Wireframe Gate (RELEASE GATE)
The following wireframe states required in WFD-5126 BEFORE any POI system development may commence:
1. POI marker on map — default + tapped state
2. POI detail panel — all Phase 1 Active category types
3. POI search entry point + results list
4. POI group filter / toggle panel — all 10 display groups with toggle controls + archetype defaults indicated
5. *'Navigate here'* flow from POI detail panel
6. POI marker — glove-compatible touch target validation (min 44×44pt)
7. Empty state — no POIs in downloaded region for a given group

**No development may commence without WFD-5126 approval of all 7 states.**

### §6.2 POI Search Capability

| Capability | Specification |
|---|---|
| **Entry point** | Persistent search control on main map view. Not buried >1 tap from map. Also from POI detail flow. |
| **Search scope** | Complete Phase 1 Active POI dataset within downloaded regional bundle. **No network call required or permitted.** |
| **Search fields** | POI display name · POI category display name · POI group name. Partial string matching. E.g. 'camp' returns CAM-01..05 + any POI with 'camp' in name. |
| **Search results** | List sorted by **distance from current GPS (nearest first)**. Each result: POI name, category icon, group label, distance. Tap → opens detail panel + centres map. |
| **Ranking** | **Distance only.** No AI ranking, relevance scoring, popularity weighting, adaptive ordering. |
| **Offline operation** | Search index built from bundle at download time or first launch. No connectivity at search time. Index survives app restart. |
| **Search × group toggle interaction** | Search returns results from ALL groups regardless of toggle state. Toggled-off group still appears in search + navigable. Toggling off does NOT remove from search. |
| **No AI predictive input** | No predictive text, AI-suggested queries, or adaptive history weighting. Standard autocomplete against downloaded POI name index = permitted. |

> **NOTE — Empty state for out-of-region search:** POI search scope limited to downloaded regional bundle. POIs outside region not searchable. Vendor must implement empty state informing user + providing clear action to expand regional map download. Empty state copy = PD review as part of WFD-5126 approval. Authority: WFD-5126; UXS-5726 non-alarming language model.

### §6.3 POI Display Groups & Toggle Model

#### §6.3.1 Group-Level Toggle Model (selection rationale)

| Model | Description | Assessment |
|---|---|---|
| All on/off (binary) | Single toggle for all POIs (Hema 4WD Explorer pattern) | Too blunt for 55 categories. Cognitively overwhelming at high density. |
| **Group-level toggle (chosen)** | User toggles functional groups (Water, Fuel, Camping…). Archetype preset sets defaults. | Appropriate cognitive granularity. Matches how users think. Smarter than Hema; simpler than GAIA GPS. |
| Individual category toggle | Toggle each of 44 codes independently | Too granular for stressed field use. Phase 2 power-user candidate. |

> **Competitive note:** GAIA GPS (primary competitor for Bushwalker/backcountry) implements individual overlay toggling via multi-step Adventure Mode + layers menu. 2026 update drew user criticism for too-many-taps. trackaroo® group-level + archetype-preset defaults = deliberate competitive differentiator. Authority: CRG-5026.

#### §6.3.3 Toggle Behaviour Rules

| Rule | Specification |
|---|---|
| Archetype preset defaults | Sets group default at first launch or archetype change. Defaults must NOT be modified by telemetry/route history/adaptive logic |
| User override | User may toggle any group on/off any time. Always manual, user-initiated |
| Preference persistence | Persists across sessions without connectivity. Survives app restart |
| Search override | Returns results across all groups regardless of toggle state |
| **Emergency group lock** | Group 4 default ON for all archetypes + not removable from filter panel. User may toggle off manually but default always ON |
| **Camping group protection** | Group 3 default ON for all archetypes. No preset may default OFF |
| No auto-suggestion | System must NOT prompt user to enable group based on location/route/intent |
| No adaptive defaults | Defaults must NOT change based on telemetry/usage |

### §6.5 PCR Framework Intersection with POI Categories

**PCR markers visually + architecturally independent from POI markers.** PCR submission available at all POI locations (§6.1.4).

| PCR Category | Relevant POI Codes | Interaction | Primary Use Cases |
|---|---|---|---|
| **PCR-OBS** (Obstruction) | TRK-01, TRK-05, CAM-01..04, FHT-01 | PCR-OBS does NOT alter POI display state. Both independently visible. | Fallen tree blocking campsite access · debris on track junction · obstruction at boat ramp |
| **PCR-CLO** (Closure) | CAM-01..05, ACC-01..04 | PCR-CLO may indicate POI destination inaccessible. POI remains displayed. PCR = supplementary condition data. | Campground closed by land manager · gate locked · seasonal closure · damaged hut condemned |
| **PCR-INF** (Infrastructure Damage) | CAM-05, FAC-01..06, TRK-03, TRK-04 | Damage surfaced via PCR. POI icon unchanged. | Hut/shelter roof damaged · toilet out of service · causeway damaged |
| **PCR-SRF** (Surface Condition) | TRK-05, WAT-03, TRK-03, CAM-01..04 | Surface condition. POI display unaffected. | Boggy campsite access · flooded ford · washed-out entry road |
| **PCR-WTR** (Water Crossing) | WAT-03, TRK-03, TRK-04, FHT-01, FHT-04, CAM-01..04 | **Highest-relevance intersection.** Current crossing condition context near campsite/shelter/water access. | Flooded creek on approach to free camp · causeway underwater · boat ramp submerged or inaccessible due to flood |
| **PCR-HAZ** (Localised Hazard) | EMG-06, CAM-05, ACC-03, CAM-01..04 | Hazard at/near campsite/shelter/access. POI display unaffected. | Fire danger near campsite · dangerous wildlife · unstable terrain near hut |

## Key takeaways

- **55 Phase 1 Active POI categories** across **10 display groups**
- **Camping (CAM-01..05) + Emergency (EMG-01..07) = mandatory ON for ALL archetypes** — no preset may default OFF
- **POI = static curated reference data** (read-only from Mapbox/OSM bundle, no user-modification, no cloud sync Phase 1)
- **PCR available at ALL POI types** — single user-submitted condition mechanism (no separate "campsite condition" reporting)
- **POI and PCR are independent layers** (LIR-04, LIR-05) — must never merge icons or modify each other
- **Group-level toggle (not individual)** = Phase 1 model · Individual = Phase 2 power-user candidate
- **Search**: offline-only, distance-ranked only, no AI/relevance/popularity weighting
- **User-saved custom POIs** = covered by Safe Anchor Point (FSD §5.2) Phase 1
- **AI-powered POI search/ranking** = **prohibited forever** (Survival Core governance, UXS-5726)
- **Working ref `PCR-5026` retired-before-issue** → all references must use POI-5026
